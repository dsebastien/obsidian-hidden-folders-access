# 2026-04-15

Replaced the plugin template with the **Hidden Folders Access** implementation.

## Accomplished

- Scoped the problem: user wants hidden root-level folders (e.g. `.claude`) to be fully indexed by Obsidian (file tree, metadata cache, Bases) while keeping on-disk names intact. Symlinks and folder renames ruled out (Sync / cross-platform / external tool compatibility).
- Reverse-engineered the desktop adapter to find the hidden-filter boundary: `listRecursiveChild` and `reconcileFile` both drop hidden paths via an internal `ru(path)` check and reroute them to `reconcileDeletion`.
- Built `HiddenFoldersIndexer` in `src/app/services/`:
    - Lists hidden root folders via `adapter.list('/')` (bypasses public `exists`/`stat`, both of which apply the filter).
    - Monkey-patches `listRecursiveChild` and `reconcileFile` so whitelisted paths skip the filter and call `reconcileFileInternal` directly.
    - Cascades `reconcileFolderCreation(path, path)` to inject every descendant; uses `watchHiddenRecursive(path)` for live fs events.
    - Tears down cleanly: stops watchers, calls `reconcileDeletion` bottom-up, restores originals.
- Rewrote settings tab: lists discovered hidden folders with per-folder toggles and a rescan button.
- Updated manifest (`id: hidden-folders-access`, `isDesktopOnly: true`) and package metadata.
- Validated on the live vault (`/home/dsebastien/notesSeb`):
    - 1142 entries indexed under `.claude` (474 folders + 668 files, 518 markdown).
    - Metadata cache populated — 513/518 markdown files have parsed frontmatter.
    - Live watcher picks up create and delete events (<3s).
    - Disable removes everything cleanly, watchers drop to zero.

## Key decisions

- Keep the adapter-level patches scoped to whitelisted prefixes so unrelated hidden paths (e.g. `.obsidian`) retain the original filtering.
- Restore the original adapter methods once the last folder is disabled so there is zero overhead when the plugin is enabled but no folders are selected.

## Open questions / follow-ups

- `watchHiddenRecursive` relies on Linux/macOS fs.watch. It has known flakiness with some filesystems (network mounts). Possible fallback: poll via `listRecursive` on a schedule.
- `disablePath` awaits `reconcileDeletion` sequentially; for very large trees this is visible as a freeze. Batching / yielding could smooth the UX.
- The plugin is desktop-only. Mobile (`CapacitorAdapter`) doesn't expose the same internals; investigating whether a narrower vault-cache injection is feasible there is deferred.
